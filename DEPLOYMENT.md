# GitHub Pages Deployment Guide

## Overview
This project uses Frozen-Flask to generate a static site from a Flask application, which is then deployed to GitHub Pages from the `docs/` folder.

## Deployment Setup

### GitHub Pages Configuration
1. Go to your repository settings
2. Navigate to **Pages** section
3. Under "Build and deployment":
   - **Source**: Deploy from a branch
   - **Branch**: `main` (or your default branch)
   - **Folder**: `/ (root)` or `/docs` - Make sure to select `/docs`

### Project Structure
- **Source code**: Flask app in root directory (`app.py`, `templates/`, `static/`)
- **Static output**: Generated site in `docs/` folder (auto-generated by Frozen-Flask)
- **Configuration**: `freeze.py` contains Frozen-Flask settings

## Build & Deployment Process

### Local Development
```bash
# Install dependencies
pip install -r requirements.txt

# Run Flask development server
python app.py
```

### Generate Static Site
```bash
# Activate virtual environment (Windows)
.\.venv\Scripts\Activate

# Or on macOS/Linux
source .venv/bin/activate

# Run the freezer
python freeze.py
```

This generates the complete static site in the `docs/` folder, including:
- All HTML pages with relative paths
- CSS and JavaScript assets
- 151 Pokémon detail pages
- All section pages (about, compare, memory, pokemones, runner)

### Deploy to GitHub Pages
```bash
# Commit the generated files
git add docs/
git commit -m "Update static site"

# Push to GitHub
git push origin main
```

GitHub Pages will automatically deploy from the `docs/` folder after the push.

## Important Files

### Configuration Files
- **freeze.py**: Frozen-Flask configuration and URL generators
- **app.py**: Flask application with all routes
- **requirements.txt**: Python dependencies

### Generated Files (in docs/)
- **.nojekyll**: Tells GitHub Pages to skip Jekyll processing
- **index.html**: Main entry point (root page)
- **pokedex/index.html**: Pokédex landing page
- **pokemon/*/index.html**: Individual Pokémon detail pages (151 total)
- **static/**: CSS and JavaScript assets
- **about/, compare/, memory/, pokemones/, runner/**: Feature pages

## Path Configuration

All HTML files use **relative paths** for compatibility with GitHub Pages:
- Images and external resources use direct URLs (PokeAPI)
- CSS/JS use relative paths: `./static/css/base.css` or `../static/css/base.css`
- Navigation links use relative paths: `./`, `./pokemones/`, `../pokemon/pikachu/`

## Troubleshooting

### GitHub Pages Shows README Instead of Content
1. Verify `docs/index.html` exists and has content
2. Check that GitHub Pages source is set to `/docs` folder
3. Verify `.nojekyll` file exists in `docs/` folder
4. Clear browser cache and hard refresh (Ctrl+Shift+R or Cmd+Shift+R)

### Assets Not Loading (CSS/JS)
1. Check that relative paths in HTML files match actual file locations
2. Verify all CSS and JS files are in `docs/static/` folder
3. Check browser console for 404 errors
4. Ensure paths use `./` for same-level and `../` for parent-level access

### Pokémon Detail Pages Not Loading
1. Verify all 151 Pokémon directories exist in `docs/pokemon/`
2. Check that `freeze.py` includes Pokemon URL generator
3. Ensure each Pokémon page has correct relative paths

## Automation (Optional)

For automated deployment on every push, you can add a GitHub Actions workflow:

Create `.github/workflows/deploy.yml`:
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: 3.9
      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
      - name: Generate static site
        run: python freeze.py
      - name: Commit and push
        run: |
          git config --local user.email "action@github.com"
          git config --local user.name "GitHub Action"
          git add docs/
          git commit -m "Auto-update static site" || true
          git push
```
